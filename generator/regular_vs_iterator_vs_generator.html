Three-way comparison of regular functions, iterator classes, and generator functions in Python.

Same task, three ways
Produce the numbers 1, 2, 3, ..., n
Regular function
Uses return
def count_up_to(n): result = [] i = 0 while i < n: i += 1 result.append(i) return result # call once, get all nums = count_up_to(3) # nums = [1, 2, 3]
Runs All at once
Memory Holds all n
Code effort Low
Infinite ok? No
Iterator class
Uses __next__
class CountUpTo: def __init__(self, n): self.n = n self.i = 0 def __iter__(self): return self def __next__(self): if self.i >= self.n: raise StopIteration self.i += 1 return self.i
Runs One at a time
Memory Tiny
Code effort High
Infinite ok? Yes
Generator function
Uses yield
def count_up_to(n): i = 0 while i < n: i += 1 yield i # pauses at each yield gen = count_up_to(3) for x in gen: print(x) # 1, 2, 3
Runs One at a time
Memory Tiny
Code effort Low
Infinite ok? Yes

Situation Best choice Why
Small, fixed-size result (a few dozen items) Regular Simplest; caller gets a real list
Huge or infinite stream Generator Lazy, tiny memory, shortest code
Reading a big file line by line Generator Same reason
Need reset, peek, or extra methods on the iterator Iterator class You control the object's API
Need to inspect or modify iterator state from outside Iterator class Attributes are accessible; a generator's frame is not
90% of everyday "produce a sequence" tasks Generator Best default in modern Python
